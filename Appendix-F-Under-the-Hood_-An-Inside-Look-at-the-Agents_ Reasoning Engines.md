# 부록 F. Under the Hood: Agents' Reasoning Engines(추론 엔진 내부)

## F.1 개요

- **프롬프트**: "추론할 수 있다는 건 알겠는데, 어떻게 추론하는지 단계별로 설명해 주세요."
- **목적**: **LLM 내부 메커니즘** 분석(Gemini·ChatGPT·Grok 등).
- **공통**: 패턴 인식·패턴 매칭·다단계(분해→검색→합성→생성).

---

## F.2 주요 LLM 추론 과정

| LLM | 핵심 단계 |
|-----|-----------|
| **Gemini** | 1. 프롬프트 해체 2. 정보 검색·합성 3. 응답 생성 (패턴 매칭) |
| **ChatGPT** | 1. 입력 이해 2. 지식 활성화 3. 추론 선택(CoT) 4. 시뮬레이션 5. 작성 6. 톤 조정 |
| **Grok** | 1. 쿼리 이해 2. 맥락 검색 3. 모델 공식화 4. 옵션 평가 5. 응답 구성 6. 검토 (반복 개선) |
| **Kimi** | 1. 입력 구문 분석 2. 검색 3. 구조 식별 4. 사고 사슬 5. 확률 생성 6. 반복 7. RLHF 적용 |
| **Claude** | 패턴 인식 → 검색 → 구조 계획 → 반복 개선 → 제약 검사 (직관적) |
| **DeepSeek** | 입력 토큰화 → 구문 분석 → 검색 → 전략 선택 → 실행·검증 → 생성 (계산적) |

---

## F.3 공통 패턴

- **1. 입력 분석**: 의도·유형 파악.
- **2. 지식 검색**: 훈련 데이터 패턴 활성화.
- **3. 구조화**: 단계 계획(CoT).
- **4. 생성·평가**: 토큰 예측·반복 개선.
- **5. 출력**: 형식·톤 적용.

**한계**: 인간 직관 없음, 훈련 데이터 의존, 확률적.

---

## F.4 에이전트 함의

- **역할**: **중앙 추론 엔진**(목표 분해·계획·실행).
- **강점**: 논리 시뮬레이션(도구·환경 상호작용).
- **미래**: 신뢰성 ↑ → 고도 에이전트.

---

## F.5 핵심 정리

- **공통 프레임워크**: **분해→검색→합성→생성**.
- **에이전트 핵심**: LLM = "마인드".
- **통찰**: 패턴 시뮬 → 인간 유사 추론.
