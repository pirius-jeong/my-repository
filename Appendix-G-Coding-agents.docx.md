# 부록 G. Coding Agents(코딩 에이전트)

## G.1 바이브 코딩 → 팀원 전환

- **바이브 코딩**: **초기 프로토타입·아이디어 창출**(빈 페이지 극복).
- **한계**: 프로덕션 코드 부족 → **전문 에이전트 팀**.
- **트렌드**: Google/MS "코드 30% AI 생성" (속도 ↑).

---

## G.2 인간-에이전트 팀 원칙

1. **인간 오케스트레이션**: 개발자 = 리더(목표·결정).
2. **맥락 우선**: 코드베이스·문서·설명 브리핑.
3. **직접 프론티어 모델**: Gemini 2.5·Claude 4 등.

---

## G.3 전문 에이전트 팀

| 에이전트 | 역할 | 프롬프트 예 |
|----------|------|-------------|
| **Scaffold (구현자)** | 새 코드 작성 | "01_BRIEF.md 기반 구현" |
| **Test Engineer (QA)** | 단위·통합 테스트[pytest] | "예외 처리·테스트 철학" |
| **Documenter (서기관)** | API 문서(마크다운) | "요청/응답 예시" |
| **Optimizer (리팩토링)** | 성능·가독성 개선 | "병목 분석·제안" |
| **Reviewer (감독자)** | 비평 → 반성 요약 | "우선순위 피드백" |

---

## G.4 구현 체크리스트

- **모델 액세스**: 2+ 프론티어 API 키.
- **컨텍스트 오케스트레이터**: context.toml (파일 컴파일).
- **프롬프트 라이브러리**: /prompts/ Git 관리.
- **Git Hooks**: 커밋 전 자동 리뷰.

---

## G.5 리더 원칙

- **소유권**: 아키텍처 "무엇·왜".
- **브리핑**: 모호함 제거.
- **게이트**: 최종 검증.
- **대화**: 반복 피드백.

---

## G.6 핵심 정리

- **팀**: 인간 리더 + **전문 에이전트**.
- **혜택**: 창의 ↑, 속도 ↑, 품질 유지.
- **미래**: **확장된 개발자**.
