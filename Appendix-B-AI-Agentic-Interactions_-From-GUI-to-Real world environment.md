# 부록 B. AI Agentic Interactions: GUI to Real-world(에이전트 상호작용: GUI → 실세계)

## B.1 개요

- **주제**: 에이전트가 **디지털(GUI)·물리 환경** 능동 상호작용.
- **진화**: 대화 → GUI 제어 → 실세계 인식·행동.
- **단계**: 시각 인식 → 요소 식별 → 해석 → 행동·반응.

---

## B.2 컴퓨터 상호작용 (ACI)

### 핵심 프로젝트

| 프로젝트 | 설명 |
|----------|------|
| **ChatGPT Operator (OpenAI)** | 데스크톱 앱 자동화(CRM·여행 예약) |
| **Project Mariner (Google)** | Chrome 에이전트(아파트 검색·필터) |
| **Claude Computer Use (Anthropic)** | 데스크톱 제어(PDF→스프레드시트→이메일) |
| **Browser-Use** | DOM 제어(추출·폼·탐색) |

---

## B.3 환경 상호작용

### 핵심 프로젝트

| 프로젝트 | 설명 |
|----------|------|
| **Project Astra (Google)** | 카메라·마이크: 물건 찾기·코드 디버그 |
| **Gemini Live (Google)** | 실시간 음성·카메라·화면 공유 |
| **GPT-4o (OpenAI)** | 음성·시각·텍스트 오미(저지연) |
| **Seeing AI (Microsoft)** | 시각 장애인 지원(사물·텍스트 인식) |

---

## B.4 바이브 코딩

- **개념**: **직관·대화 개발**(목표 "바이브" 제시 → AI 코드).
- **특징**: 반복 피드백·메모리 뱅크(스타일 저장).
- **혜택**: 구현 대신 창의 집중, 혁신 ↑.

---

## B.5 한눈에 보기

- **무엇**: **GUI 제어 → 실세계 다중모드**.
- **왜**: 인간 수준 작업 자동화.
- **어떻게**: 스크린샷·LLM 해석·마우스/키보드 + 카메라/마이크.
- **언제**: 데스크톱·웹·물리 작업.

---

## B.6 핵심 정리

- 에이전트 상호작용 = **디지털·물리 융합**.
- Operator·Astra 등으로 범용 비서 실현.
- 바이브 코딩: 개발 패러다임 변화.
