# 21장. Exploration & Discovery(탐험과 발견)

## 21.1 패턴 개요

- **정의**: 에이전트가 **미지의 영역 적극 탐색**·새로운 지식 생성 패턴.
- **왜 필요?** 정적 지식 한계 → **혁신·발견**(개방 환경).
- **차이**: 반응 vs 능동(하위 목표 자율 설정).

### 핵심 과정

| 과정 | 설명 |
|------|------|
| **탐색** | 환경 스캔·실험 |
| **발견** | 가설·통찰 생성 |
| **확장** | 이해·역량 성장 |

---

## 21.2 응용 사례

| 도메인 | 예시 |
|--------|------|
| **과학** | 가설·실험 자동화 |

## 21.3 Google AI Co-Scientist

- **다중 에이전트**: 생성·반성·랭킹(Elo)·진화·근접·메타 리뷰.
- **흐름**: 가설 생성 → 토론 → 개선(테스트 시간 스케일링).
- **검증**: GPQA 78.4% / 실험(AML 약물·간 섬유증).

---

## 21.4 Agent Laboratory

- **워크플로**: 문헌 검토 → 실험(코드 실행) → 보고서(LaTeX).
- **에이전트 계층**: 교수(지도)·포닥(실행)·리뷰어(판단)·ML/SW 엔지니어.
- **저장소**: AgentRxiv(공유).

### 리뷰어 예제

```python
class ReviewerAgent:
    def inference(self, plan, report):
        review_1 = get_score(plan, report, reviewer_type="엄격 연구")
        review_2 = get_score(plan, report, reviewer_type="영향력 중점")
        review_3 = get_score(plan, report, reviewer_type="혁신 중점")
        return f"리뷰어 #1:\n{review_1}\n리뷰어 #2:\n{review_2}\n리뷰어 #3:\n{review_3}"
```

## 21.5 한눈에 보기

- **무엇**: **능동 탐색**(가설·실험·발견).
- **왜**: 혁신 가속(인간 증강).
- **어떻게**: 다중 에이전트(Co-Scientist·Lab), 반복 루프.
- **언제**: 미해결·진화 영역.

---

## 21.6 핵심 정리

- 탐험/발견은 **에이전트 진화** 핵심.
- Co-Scientist처럼 안전·인간 루프 결합.
- 책 완결: 완전 에이전트 시스템.
